1. Executive Overview

BkonnectHomes is a robust, full-stack web application designed to bridge the gap between university students searching for safe accommodation and property owners looking to manage their hostel listings.

Primary Objective: Eliminate the manual, time-consuming process of hostel hunting by providing a centralized digital ledger for discovery, side-by-side comparison, and instant bookings.

1.1 Technology Stack

  • Frontend Layer: React.js, Vite, TailwindCSS (for responsive, premium styling), Lucide-React (iconography).
  • Backend Layer: Node.js, Express.js (REST API).
  • Database Layer: Prisma ORM connected to SQLite (Development) / PostgreSQL (Production).
  • Security: JSON Web Tokens (JWT) for stateless authentication and bcrypt for password encryption.

3. Role-Based Capabilities

Role Permissions & System Capabilities
STUDENT Can browse properties, filter by "University" or "Private", use the comparison engine, favourite hostels, and place bookings. Dashboards track booking timestamps and statuses.
OWNER Can register properties, upload base64 images, define room types (Single/Double), set monthly rent, and view incoming bookings for their specific properties.
ADMIN A super-user account that accesses the Master Booking Ledger to oversee all transactions, monitor total platform metrics, and verify data integrity.

4. Core Technical Features

4.1 The Comparison Engine

A completely custom React Context (`CompareContext`) interacts with the browser's `localStorage` to create a persistent comparison queue. Up to 3 properties can be compared side-by-side, where the system dynamically calculates intersectional amenities (e.g., automatically detecting if all 3 hostels offer Wi-Fi) and renders starting prices.

4.2 Booking Timestamps & Ledger

All bookings utilize Prisma's `createdAt` schema to generate precise timestamp logs. These timestamps are formatted on the frontend to provide both the Admin and the Student with an exact audit trail of when reservations were initiated.

4.3 Automated Fallback Imagery

To guarantee a premium UI experience, the system utilizes a robust `onError` failsafe pattern for all property imagery. If an external URL fails or a CDN drops a request, the React components automatically intercept the error and inject a curated, high-quality, authentic Kampala student dorm image to maintain visual harmony.

5. Deployment Protocol

The system is packaged and prepared for cloud deployment using standard configuration files (`vercel.json` and `render.yaml`).

  • Database Migration: A pre-deployment script (`fix_categories.js`) ensures all legacy data is normalized strictly into binary classifications (Private vs. University).
  • Backend Hosting: The Node.js Express server is built to deploy on Render.com, exposing RESTful API endpoints.
  • Frontend Edge Hosting: The React/Vite build is optimized for Vercel, ensuring fast global content delivery for the UI.
